CU CS Standard Development Environment

As part of the CU CS Foundation Program, the department develops and supports the use of a standard development environment (SDE) for all CS courses. The goal of this environment is to provide all CS students with the tools they need to complete their CS courses and to prepare students for CS development work in the real world.

The standard development environment is provided in several flavors. Students are encouraged to use whichever flavor they feel best fits their preferences and requirements.

Getting Help

If you require extra help with the VM or other component of the SDE, stop by any of the CSEL AMA office hours or andrew.sayler@colorado.edu.

CU CS Virtual Machine

The simplest method for getting up and running with the SDE is to use the CU CS Virtual Machines. A Virtual Machine is a simulated "computer" that runs as a program on your real computer. The CU CS VM comes pre-installed with a copy of the SDE and is ready to use out of the box for the following courses:

  • CSCI 1300
  • CSCI 2270
  • CSCI 2400

The CU CS VM can also be used with a number of upper level CS courses. Simply setup the VM via the steps below, and then install the required package for your specific course by opening a terminal on the VM and running:

sudo apt-get update; sudo apt-get install <course package>

To setup your copy of the VM, follow the steps below:

CU CS Apt Packages

Students wishing to set up native Linux systems running the CU SDE (i.e. not use the VM) may do so by using the packages hosted in the CU CS Apt repository. Each CU core course (and many of the optional courses) have associated Apt packages that will automatically install all required software for the course in a single, simple, step.

Currently, we provide packages for Ubuntu 12.04 LTS (Precise Pangolin). These packages may also work with other versions of Ubuntu or other Linux distributions supporting the apt/deb package format, but YMMV.

To use and install the CU CS Apt packages, follow the directions at apt.cs.colorado.edu.

CU CS Education Laboratory (CSEL)

The Computer Science Education Laboratory (CSEL) is a CU Linux lab that runs a copy of the CU CS SDE. The lab consists of both physical machines that students can use in person, as well as remote serves that students may SSH into to do work. The CSEL is available for use by any student enrolled in at least one CU CS course.

More information on the CSEL is available at csel.cs.colorado.edu.
