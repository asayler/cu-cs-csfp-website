Obtaining the VM Image

The recommended VM Image download method is via the BitTorrent peer-to-peer distribution system. For those that can not use BitTorrent, we also provide a direct download method. Note, however, that the direct download method is slower and more prone to corruption than the BitTorrent method.

We also includes a link for the md5 hash file associated with the VM image. You can (optionally) use this hash file with the Unix 'md5sum' utility to verify that your download is not corrupt. See this link for more information.

Older versions of both the VM image (both torrent and direct downloads) can be found on the CSEL Archive site.

Download via BitTorrent

Downloading the VM via BitTorrent involves downloading a '.torrent' file that you can then open with a BitTorrent client to download the actual VM file. Using BitTorrent will help ensure you the fastest download, will ensure your download is not corrupted, and will allow you to easily start, stop, a resume your download should you need to. If you do not already have a BitTorrent client installed, please install one from the list below:

  • FrostWire (OSX + Windows + Linux, Free)
  • qTorrent (OSX + Windows + Linux, Free)
  • Transmission (Linux + (experimental) OSX, Free)

Once you have installed a BitTorrent client, you may download the one of the .torrent files for the VM below. Open the .torrent file in your BitTorrent client to begin your VM download.

VM .torrent:

  • CU-CS-VM-Fall13-x64-v1.3.ova.torrent

VM md5 verification .torrent (optional):

  • CU-CS-VM-Fall13-x64-v1.3.ova.md5.torrent

Direct Download

If you can not use BitTorrent to download the VM, you may instead use one of the direct download links below. Please use BitTorrent if possible as it will ensure a faster, more reliable download.
