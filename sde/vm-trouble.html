Troubleshooting the VM

Having trouble with the VM? See the items below. You may also wish to consult the VM Tips page.

VirtualBox 64-bit VT-x/AMD-V "Not Operational" Error

If you are using a computer on which the processor virtualization extensions (VT-x/AMD-V) have been disabled by default (or if you have manually disabled them), you will receive the following error message when booting any 64-bit VM:

VT-x/AMD-V hardware acceleration has been enabled, but is not operational. Your 64-bit guest will fail to detect a 64-bit CPU and will not be able to boot.

Please ensure that you have enabled VT-x/AMD-V properly in the BIOS of your host computer.

The solution to this problem is to reboot your host computer, enter the BIOS/System setup screen before your host OS starts, and enable processor virtualization support. The exact process to do this differs by computer model, but the general process for a PC is listed below. In general, Mac/OSX user will not encounter this issue.

  1. Reboot the host computer
  2. As soon as the host computer begins to boot (before the host OS starts) look for a message on the screen telling which button to push to enter Setup (often it's F1, F2, Enter, Esc, or F8). Push this button quickly to launch the BIOS setup menu.
  3. Find the virtualization settings in the setup menus. Often they are located with the "Security" settings.
  4. Change all virtualization settings (especially the VT-x or "Virtualization" setting) from "Disabled" to "Enabled".
  5. Save your BIOS settings and exit/reboot

If you can not enable virtualization, if you processor does not support virtualization, or if you are using a 32-bit processor or a 32-bit OS you may need to fall back to using the 32-bit VM image. Please contact your instructor or CSFP personnel for more information.
